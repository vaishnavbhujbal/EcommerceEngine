# CLAUDE.md — Discovery Engine

## What we're building
An AI-powered ecommerce discovery engine (a POC). It crawls a store with a headless
browser, structures the catalog with AEO + GEO, embeds it, and answers shopper prompts
with grounded, conversion-oriented responses (live price/stock, citations, cross-sell,
upsell) — proving it beats generic ChatGPT/Perplexity answers for a specific catalog.

Full spec lives in `docs/POC_Technical_Design.md`. Setup steps in `docs/SETUP.md`.

## Stack
- Backend: Node.js + Express + TypeScript (run with tsx)
- Frontend: React + TypeScript + Vite + Tailwind v4
- Headless browser: Playwright (chromium)
- Database: Supabase (Postgres + pgvector) — connect via DATABASE_URL connection string
- LLM + embeddings: OpenAI (GPT-4o + text-embedding-3-small), isolated behind `ai.ts`

## Golden rules (do not break these)
1. **Never invent catalog facts.** Price, SKU, and stock come ONLY from SQL/retrieval —
   never generated by the model. If unknown, say so.
2. **Semantic data in vectors; volatile data in SQL.** Embed descriptions/reviews/FAQs/
   specs/GEO content. NEVER embed price or stock — keep those as SQL columns joined at
   answer-time so answers are always current.
3. **All model calls go through `ai.ts`** (`embed()` and `complete()`), so we can swap
   providers without touching the pipeline.
4. **Keep it simple.** Flat files, one concern each. No premature abstractions or folders.

## Build order (one file at a time)
Backend `src/`:
1. `db.ts` — pg pool
2. `ai.ts` — embed() + complete()
3. `index.ts` — Express app + /health (verify DB connection works first)
4. `migrations/001_init.sql` — schema from design doc §6 (incl. related_products)
5. `crawler.ts` — Playwright: render a PDP, extract JSON-LD + DOM
6. `ingest.ts` — crawl → AEO/GEO → embed → store
7. `retrieve.ts` — hybrid retrieval (vector + SQL filters + live price/stock join)
8. `rag.ts` — intent router → ground → GPT-4o → recommendation engine
9. `routes.ts` — POST /ingest (SSE), POST /chat (SSE)
Then frontend: wire the existing prototype's mock timers to real /ingest and /chat.
